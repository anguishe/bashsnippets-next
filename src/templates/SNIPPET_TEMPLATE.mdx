## What Breaks Without This Script

When `/var/log` fills silently, nginx stops writing access logs, MySQL cannot flush transactions, and your next deploy fails mid-write — with no alert until users report errors. A full disk does not send a polite warning; it corrupts in-progress work and takes down every service that still needs to write bytes. This script reads usage with `df`, compares it against a threshold you control, and exits non-zero when you are running out of room — so cron or your monitoring stack can act before the crash.

## The Script

Save as `your-script.sh`. Change `THRESHOLD` and `TARGET` at the top before running on production.

```bash
#!/bin/bash
# Script: your-script.sh
# Purpose: Alert when disk usage exceeds a threshold — prevent full-disk service crashes
# Usage: ./your-script.sh [THRESHOLD_%]  (default: 80)
# Tested: Ubuntu 22.04 LTS, Debian 12, macOS Ventura
set -euo pipefail

# ── CONFIGURATION ──────────────────────────────────────────────
THRESHOLD="${1:-80}"            # Usage % that triggers the alert (override via arg)
TARGET="/"                      # Mount point to check — use /var on web servers with large logs

# ── CONSTANTS ──────────────────────────────────────────────────
CHECK="✓"
CROSS="✗"

# ── MAIN ───────────────────────────────────────────────────────
USAGE=$(df "$TARGET" | awk 'NR==2 {print $5}' | tr -d '%')

if [ "$USAGE" -ge "$THRESHOLD" ]; then
  echo "$CROSS Disk at ${USAGE}% on $TARGET — threshold is ${THRESHOLD}%"
  exit 1
fi

echo "$CHECK Disk at ${USAGE}% on $TARGET — within ${THRESHOLD}% threshold"
```

<Callout type="success" title="What this does, line by line">

- **`THRESHOLD="${1:-80}"`** — Default 80%; pass a number as the first argument to override without editing the file.
- **`TARGET="/"`** — Which mount point `df` inspects. Match this to where data actually grows.
- **`awk 'NR==2 {print $5}'`** — Skips the header row and reads the Use% column.
- **`tr -d '%'`** — Strips `%` so bash can compare integers in `[ "$USAGE" -ge "$THRESHOLD" ]`.
- **`exit 1` on breach** — Non-zero exit lets cron mail you or triggers downstream alerting.

</Callout>

## Step-by-Step Setup

### Step 1: Create the script file

```bash
nano your-script.sh
```

Paste the script above. Save with **Ctrl+X → Y → Enter**.

### Step 2: Set your threshold and target

Edit `THRESHOLD=` and `TARGET=` at the top. On a web server, `TARGET="/var"` often catches log growth before root fills.

### Step 3: Make it executable

```bash
chmod +x your-script.sh
```

### Step 4: Run it manually

```bash
./your-script.sh
```

**When usage is below the threshold:**

```text
✓ Disk at 42% on / — within 80% threshold
```

**When usage is above the threshold:**

```text
✗ Disk at 87% on / — threshold is 80%
```

## Schedule It with Cron

Open your crontab:

```bash
crontab -e
```

Add a line with the **full path** to your script:

```bash
# Every hour
0 * * * * /home/you/your-script.sh >> /var/log/diskcheck.log 2>&1
```

## Frequently Asked Questions

### How do I run this script?

Save as `your-script.sh`, run `chmod +x your-script.sh`, then `./your-script.sh`. Exit code 1 means the check failed.

### Does this work on macOS?

Yes. Use `df -P "$TARGET"` for POSIX output if `awk` reads the wrong column on macOS.

### How do I schedule this with cron?

Run `crontab -e` and add `0 * * * * /full/path/to/your-script.sh`. Always use absolute paths in cron.
