SECURITY RISK: 777 Permissions

chmod 777 gives EVERYONE full read/write/execute access. Never use this on production servers or sensitive files. Anyone can modify or delete the file.

⚠️
WARNING: 666 Permissions

chmod 666 gives everyone read/write access (no execute). Avoid this on config files, logs, or any file containing sensitive data.

✓
SAFE: Standard File Permissions

chmod 644 is the standard permission for regular files. Owner can read/write, everyone else can only read. This is secure for most use cases.

⚠
WARNING: World-Writable Permission

The "Others: Write" bit is set. Any user on this system can modify or delete this file. Restrict write access to owner and group unless you have a specific reason for world-writable permissions.

Understanding chmod Permission Notation

Octal chmod uses three digits — owner, group, others — each built from read (4), write (2), and execute (1). A 755 means 7 for the owner (4+2+1: full control), 5 for group (4+1: read and execute), and 5 for others. Common values: 644 for config files (owner writes, everyone reads), 600 for secrets (only owner reads/writes), 700 for private scripts (only owner can run), and 755 for executables and directories the web server or other users need to traverse.

Symbolic notation changes specific bits without recalculating the whole number: chmod +x script.sh adds execute, u+rw gives the owner read and write, go-w strips write from group and others. Use octal when you know the final state; use symbolic when you are toggling one bit on an existing file. Set 777 on a script and anyone on the box can overwrite it — a common path to backdoors. Leave SSH private keys at 600 or ssh refuses to connect. Default deploy scripts and web roots to 755 unless you have a reason not to.

Common chmod Examples

chmod 755 deploy.sh
chmod 600 ~/.ssh/id_rsa
chmod 644 config.txt
chmod +x myscript.sh
chmod -R 755 /var/www/html
chmod 700 ~/private/

Frequently Asked Questions

What does chmod 755 mean?

Owner gets read, write, and execute. Group and others get read and execute — no write. That is the standard for shell scripts, binaries, and directories that need to be listed and entered but not modified by other users.

What is the difference between chmod 644 and 755?

644 is for regular files: owner read/write, everyone else read-only. 755 adds execute for all three groups — required for scripts and directories. Using 644 on a deploy script means it will not run.

How do I give execute permission in bash?

Run chmod +x filename to add execute for all users, or chmod u+x filename for owner only. Without the x bit, bash returns "Permission denied" when you try to run the file directly.

What does chmod +x do in Linux?

chmod +x adds execute permission for owner, group, and others. It does not change read or write bits. Use it when a script has the right content but fails with "Permission denied" on ./script.sh.
