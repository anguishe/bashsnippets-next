Common Bash Exit Codes Reference

      

        
          

            Code
            Meaning
            Common Cause
          

        
        
          

            0
            Success
            Command completed without errors
          

          

            1
            General error
            Catchall for unspecified failures — permission denied, missing file, validation error
          

          

            2
            Misuse of shell builtin
            Missing keyword, wrong argument type, or syntax error in a builtin
          

          

            126
            Command cannot execute
            File exists but lacks execute permission or is not a valid binary
          

          

            127
            Command not found
            Typo in command name, binary not installed, or not in PATH
          

          

            128
            Invalid argument to exit
            exit called with a non-integer or out-of-range value
          

          

            130
            Script terminated by Ctrl-C
            User sent SIGINT — manual interrupt during execution
          

          

            137
            Script terminated by SIGKILL
            kill -9 or OOM killer — process cannot be caught or cleaned up gracefully
          

          

            139
            Segmentation fault
            Invalid memory access — usually a crash in a compiled binary
          

          

            255
            Exit status out of range
            exit called with a value above 255 or negative — wraps to 255
          

        
      


      
Every command sets $? when it finishes. Test it in conditionals: if mycommand; then echo ok; else echo "failed with $?"; fi. With set -e, any non-zero exit stops the script immediately — which is what you want in production until you explicitly handle a known failure. Capture the code right after the command; the next line overwrites $?.

      
trap runs cleanup when a script exits for any reason — success, error, or signal. Register trap cleanup EXIT to remove temp files and release locks even after a non-zero exit. Combine with trap 'echo "failed at line $LINENO with exit $?"' ERR under set -e to log exactly where the script broke before cleanup runs.


      
Frequently Asked Questions


      
What does exit code 1 mean in bash?

      
Exit code 1 is a generic failure — the command did not succeed, but the exact reason depends on what ran. grep returns 1 when nothing matches; most other tools use it for any unhandled error. Treat it as "something went wrong" and check stderr or re-run with verbose flags.


      
How do I check the exit code of a command in bash?

      
Run the command, then immediately check echo $?. Capture it in a variable if you need it later: status=$?; if [ $status -ne 0 ]; then echo "failed with $status"; fi. $? resets after every command, so read it before running anything else.


      
What is exit code 127 in Linux?

      
Exit code 127 means command not found. Bash searched every directory in PATH and could not locate the binary. Check for typos, missing packages, or a PATH that does not include the install location.
